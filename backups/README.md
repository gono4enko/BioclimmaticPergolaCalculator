# Инструкции по восстановлению из резервных копий

В этой директории хранятся резервные копии всех файлов проекта для разных версий.

## Структура резервных копий для версии 3.8

- `app_basic_v3.8_stable.py` - основной файл приложения
- `pergola_descriptions_v3.8_stable.py` - файл с описаниями типов пергол
- `pdf_generator_fpdf_rus_v3.8_stable.py` - генератор PDF-документов
- `VERSION_v3.8.txt` - информация о версии
- `config/v3.8/` - директория с конфигурационными файлами

## Процесс восстановления

Для восстановления из резервной копии выполните следующие шаги:

1. Скопируйте файл `app_basic_v3.8_stable.py` на место основного файла приложения:
   ```bash
   cp backups/app_basic_v3.8_stable.py app_basic.py
   ```

2. Восстановите конфигурационные файлы:
   ```bash
   cp backups/config/v3.8/*.py config/
   ```

3. Восстановите генератор PDF:
   ```bash
   cp backups/pdf_generator_fpdf_rus_v3.8_stable.py pdf_generator_fpdf_rus.py
   ```

4. Восстановите информацию о версии:
   ```bash
   cp backups/VERSION_v3.8.txt VERSION.txt
   ```

5. Перезапустите сервер приложения.

## Версии и изменения

История изменений между версиями сохраняется в файле `CHANGELOG.txt`.

## Важные примечания

- Работа стабильных версий подтверждена проверкой всех функций.
- При серьезных сбоях рекомендуется восстанавливать полный комплект файлов одной версии, а не отдельные файлы из разных версий.
- В случае проблем с отображением интерфейса может потребоваться также перезапустить браузер или очистить кэш.
