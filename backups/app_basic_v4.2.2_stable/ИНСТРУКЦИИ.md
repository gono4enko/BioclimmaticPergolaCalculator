# ИНСТРУКЦИИ ПО РАБОТЕ С КАЛЬКУЛЯТОРОМ ПЕРГОЛ

## ВАЖНО: СОХРАНЕНИЕ ОРИГИНАЛЬНОГО КОНТЕНТА

**Категорически запрещены любые изменения текста описаний, путей к изображениям и контента**

- Все оригинальные тексты в модуле `config/pergola_descriptions.py` должны оставаться неизменными
- Пути к изображениям в словаре `PERGOLA_IMAGES` содержат оригинальные пути и не должны изменяться
- Подписи к изображениям в `PERGOLA_IMAGE_CAPTIONS` должны остаться оригинальными

## СТРУКТУРА ПРОЕКТА

1. **Основные файлы**:
   - `app_basic.py` - главное приложение калькулятора пергол
   - `config/pergola_descriptions.py` - модуль с описаниями пергол и изображениями
   - `CONTENT_POLICY.md` - политика содержимого (запрет на изменение контента)
   - `VERSION.txt` - текущая версия и описание изменений

2. **Система резервного копирования**:
   - В директории `backups/` хранятся стабильные версии приложения
   - Каждая версия имеет полную копию кода и конфигурации
   - Перед внесением изменений необходимо создать резервную копию

3. **Документация**:
   - `backups/CHANGELOG.txt` - полная история изменений по версиям
   - `ИНСТРУКЦИИ.md` - инструкции по работе с проектом
   - В каждой версии есть свои инструкции и документация

## РАБОТА С ВЕРСИЯМИ

1. При внесении изменений:
   - Обновить номер версии в `VERSION.txt`
   - Добавить описание изменений в `backups/CHANGELOG.txt`
   - Создать резервную копию в директории `backups/app_basic_vX.X.X_stable/`

2. Соглашение о версиях:
   - Первое число: масштабные изменения архитектуры
   - Второе число: функциональные улучшения
   - Третье число: исправления ошибок и мелкие улучшения

## РАЗВЕРТЫВАНИЕ И ЗАПУСК

1. Запуск приложения:
   ```
   streamlit run app_basic.py --server.port 5000 --logger.level debug
   ```

2. Конфигурация сервера:
   - Порт: 5000
   - Хост: 0.0.0.0 (доступ извне)
   - Логирование: уровень debug

## ИНСТРУКЦИИ ПО ПОДДЕРЖКЕ

1. Разрешенные модификации:
   - Исправление ошибок в коде (без изменения текстов)
   - Оптимизация работы приложения
   - Улучшение пользовательского интерфейса

2. Запрещенные действия:
   - Изменение оригинального текста описаний
   - Модификация путей к изображениям
   - Добавление собственного контента

3. В случае проблем с отображением:
   - Проверить пути к изображениям в словаре `PERGOLA_IMAGES`
   - Проверить корректность HTML-форматирования без изменения контента
   - Использовать функцию `display_formatted_description` для отображения
