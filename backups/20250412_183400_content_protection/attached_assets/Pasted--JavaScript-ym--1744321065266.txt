Мы настраиваем отправку JavaScript-события ym(...) в Яндекс.Метрику для того, чтобы фиксировать факт успешного использования калькулятора на сайте.
Когда пользователь нажимает кнопку "Рассчитать" и получает результат, мы вызываем функцию:

ym(65714473, 'reachGoal', 'calc_success');

Это сообщает системе аналитики (Яндекс.Метрике), что цель "успешный расчёт" достигнута.
Так мы можем:
* видеть, сколько человек реально использовали калькулятор;
* понимать, с каких рекламных источников пришли эти пользователи;
* оптимизировать рекламу (например, в Яндекс.Директе или Google Ads) по конверсиям;
* отслеживать эффективность изменения дизайна или логики калькулятора;
* увеличивать возврат инвестиций (ROI) в рекламу.

Что такое ym(...)?
Это команда для счётчика Яндекс.Метрики:
* ym(65714473, ...) — ID счётчика (уникальный для сайта)
* 'reachGoal' — тип действия (достижение цели)
* 'calc_success' — имя цели, которое мы заранее задаём в настройках Метрики



Цель:
Отправлять событие в Яндекс.Метрику ym(65714473, 'reachGoal', 'calc_success'), когда пользователь нажимает "Рассчитать" и получает результат.

Шаг 2: Вызов ym() после расчёта
Если расчёт идёт через fetch() или AJAX, вот пример JS:

<script>
document.getElementById('calc-form').addEventListener('submit', function(e) {
  e.preventDefault();

  const width = document.getElementById('width').value;
  const depth = document.getElementById('depth').value;

  fetch('/calculate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ width, depth })
  })
  .then(response => response.json())
  .then(data => {
    document.getElementById('result').innerText = 'Стоимость: ' + data.price + ' €';

    // Отправляем событие в Яндекс.Метрику
    ym(65714473, 'reachGoal', 'calc_success');
    console.log('Цель calc_success отправлена');
  })
  .catch(err => console.error('Ошибка:', err));
});
</script>

<form id="calc-form">

</form>
