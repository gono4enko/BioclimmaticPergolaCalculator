# Калькулятор стоимости пергол версии 4.4.0 (Модульная)

## Информация о версии

Эта версия приложения переработана с использованием модульной структуры, что делает код более организованным и поддерживаемым. Основные изменения включают:

- Реорганизацию кода в модули по функциональному признаку
- Создание пакета `components` для компонентов интерфейса
- Создание пакета `utils` для вспомогательных функций
- Создание пакета `config` для конфигурационных данных
- Улучшение системы логирования

## Структура проекта

```
.
├── app_basic.py             # Основной файл приложения
├── components/              # Пакет компонентов интерфейса
│   ├── __init__.py
│   ├── dimensions_form.py   # Форма ввода размеров
│   ├── header.py            # Заголовок приложения
│   ├── options_form.py      # Форма выбора опций
│   ├── orange_button.py     # Стилизованная кнопка
│   ├── results_new.py       # Отображение результатов
│   └── scroll_helper.py     # Скрипты для прокрутки
├── config/                  # Пакет конфигурации
│   ├── __init__.py
│   ├── pergola_descriptions.py  # Описания пергол
│   ├── pergola_types.py     # Типы пергол
│   └── price_data.py        # Данные о ценах
├── utils/                   # Пакет утилит
│   ├── __init__.py
│   ├── calculator.py        # Модуль расчетов
│   ├── logger.py            # Модуль логирования
│   └── price_loader.py      # Загрузка данных о ценах
└── VERSION.txt              # Файл с версией
```

## Как использовать этот бэкап

Если вам нужно восстановить эту версию приложения, выполните следующие шаги:

1. Скопируйте содержимое папки `backups/app_v4.4.0_modular` в корневую директорию проекта:
   ```
   cp -r backups/app_v4.4.0_modular/* ./
   ```

2. Убедитесь, что есть директория `logs` для хранения файлов журнала:
   ```
   mkdir -p logs
   ```

3. Запустите приложение с помощью Streamlit:
   ```
   streamlit run app_basic.py --server.port 5000
   ```

## Возможности и преимущества

Новая модульная структура обеспечивает:

1. **Улучшенную поддерживаемость** - каждый компонент находится в отдельном файле
2. **Лучшую читаемость** - логические блоки разделены по функциональному назначению
3. **Простоту расширения** - новые компоненты можно добавлять независимо
4. **Более эффективное логирование** - структурированная система логов в директории logs
5. **Удобство обновления** - можно обновлять отдельные компоненты без изменения всего приложения

## Особенности реализации

- Компоненты интерфейса полностью отделены от бизнес-логики
- Расчеты выполняются в отдельном модуле calculator.py
- Логирование настроено в модуле logger.py с сохранением логов в директории logs
- Загрузка данных о ценах вынесена в отдельный модуль price_loader.py

## Известные ограничения

- Предупреждения LSP о нерешенных импортах не влияют на работу приложения
- Некоторые функции в модулях могут быть не использованы в текущей версии, но сохранены для обратной совместимости
