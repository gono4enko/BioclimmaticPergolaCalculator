# Инструкции по восстановлению приложения из бэкапа

## Доступные версии бэкапов

В директории `backups` находятся следующие версии приложения:

- `app_v4.4.0_modular` - Модульная версия (в разработке) (10.04.2025)
- `app_basic_v4.3.0_large_headers` - **Текущая версия** с улучшенными заголовками (10.04.2025)
- `app_basic_v4.2.2_stable` - Стабильная версия с исправленными путями изображений (10.04.2025)
- `app_basic_v4.2.1_stable` - Версия с исправлениями HTML отображения (09.04.2025)
- `app_basic_v4.2_stable` - Версия с синим фирменным оформлением (08.04.2025)
- `app_basic_v4.1_stable` - Версия с исправлениями текстов и PDF генератора (05.04.2025)
- `app_basic_v4.0_stable` - Базовая версия с обновленным дизайном (01.04.2025)

## Инструкции по восстановлению

Чтобы восстановить приложение из бэкапа, следуйте этим инструкциям:

### 1. Восстановление текущей версии (4.3.0 с улучшенными заголовками)

```bash
# Скопировать файлы из бэкапа в корневую директорию проекта
cp -r backups/app_basic_v4.3.0_large_headers/* ./

# Перезапустить сервер
```

### 2. Восстановление модульной версии (4.4.0 в разработке)

```bash
# Скопировать все файлы из бэкапа в корневую директорию проекта
cp -r backups/app_v4.4.0_modular/* ./

# Создать директорию для логов, если она отсутствует
mkdir -p logs

# Перезапустить сервер
```

### 3. Восстановление стабильной версии 4.2.2

```bash
# Скопировать файлы из бэкапа в корневую директорию проекта
cp -r backups/app_basic_v4.2.2_stable/* ./

# Перезапустить сервер
```

## Возможные проблемы и их решения

1. **Проблема с импортом модулей**

   Если после восстановления возникают ошибки импорта, убедитесь, что структура директорий соответствует версии:
   - Для версий 4.4.0 и выше нужны папки `components`, `utils` и `config`
   - Для более ранних версий эти папки не требуются

2. **Проблема с отображением описаний пергол**

   Если описания отображаются некорректно (виден HTML-код), проверьте наличие и работоспособность функции:
   ```python
   def display_formatted_description(description_text):
       # ...код функции...
   ```

3. **Ошибки с прайс-листами**

   Убедитесь, что в директории `attached_assets` присутствуют все необходимые CSV-файлы с ценами:
   - Price_B500-200.csv / Price_B500-250.csv
   - Price_B700-200.csv / Price_B700-250.csv
   - Price_B600_PIR.csv

## Важные примечания

- При восстановлении из бэкапа все текущие изменения будут перезаписаны
- Рекомендуется сначала создать резервную копию текущего состояния
- Полный список изменений можно найти в файле `backups/CHANGELOG.txt`
