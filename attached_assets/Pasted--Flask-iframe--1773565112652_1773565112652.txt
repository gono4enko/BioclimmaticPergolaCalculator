Реализация экрана загрузки (загрузочный слайдбар) в Flask + iframe — разбор проблем и решений

Задача: При загрузке Flask-приложения, встроенного через <iframe> на внешний сайт, посетитель видел белый экран. Нужно было добавить экран загрузки с анимированной полосой.

Проблема 1: Глобальный CSS-сброс сломал форму

Я добавил в критический CSS в <head> стандартный reset:

*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
Это распространённая практика, но она конфликтует с Bootstrap, который уже имеет собственные базовые стили. Результат — часть элементов управления (в данном случае слайдбар загрузки) визуально ломалась.

Решение: убрать * reset полностью. Оставить в критическом CSS только стили тех конкретных элементов, которые нужны до загрузки внешних CSS-файлов (фон страницы, шапка, сам лоадер).

Проблема 2: defer на Bootstrap JS сломал инициализацию компонентов

Я добавил defer к тегу Bootstrap JS, чтобы он не блокировал рендеринг:

<script src="bootstrap.bundle.min.js" defer></script>
Но скрипты приложения (calculator.js, ui.js и др.) стоят после Bootstrap в HTML и сразу инициализируют Bootstrap-компоненты (tooltips, modals и т.д.). С defer Bootstrap выполняется после завершения парсинга HTML, но другие скрипты без defer выполняются по порядку — и к моменту их запуска Bootstrap ещё не был загружен.

Решение: убрать defer. Когда <script> теги уже стоят в конце <body>, defer не даёт никакого выигрыша — они и так не блокируют рендеринг видимой части страницы.

Что сработало без проблем:

HTML лоадера — первым элементом после <body>, чтобы браузер отрисовал его немедленно
CSS лоадера — встроен inline в <head>, поэтому применяется до загрузки любых внешних файлов
JS для скрытия — в конце <body>, слушает событие window load + страховочный setTimeout(hideLoader, 5000)
Gzip-сжатие через flask-compress — добавляется в app.py тремя строками, не требует изменений в шаблонах
<link rel="preload"> для ключевых JS/CSS — безопасно, не ломает ничего
Итоговое правило для других агентов:

При добавлении экрана загрузки в проект с Bootstrap — не трогай глобальный CSS-reset и не ставь defer на Bootstrap JS, если после него идут зависимые скрипты без defer.
