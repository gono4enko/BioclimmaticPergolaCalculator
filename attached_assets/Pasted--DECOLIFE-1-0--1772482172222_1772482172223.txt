## 1. Общее описание приложения

### Назначение
Веб-калькулятор для расчета стоимости тентовых пергол различных типов. Позволяет пользователю:
- Выбрать тип конструкции (пристенная, отдельностоящая, двухскатная, консольная FLY)
- Задать размеры (ширина, глубина)
- Выбрать модель (Project & Light, Basic & Pro New)
- Указать цвет и дополнительные опции
- Получить расчет стоимости в рублях
- Скачать коммерческое предложение в формате PDF

### Архитектурный подход
Приложение построено по принципу **монолитной веб-архитектуры**: единый Flask-сервер обслуживает как API-запросы, так и HTML-страницы с JavaScript-логикой. Это обеспечивает простоту развертывания и обслуживания.

### Целевая аудитория
- Конечные пользователи (заказчики пергол) на сайте pergolamarket.ru
- Менеджеры компании «Комфортный дом»
- Калькулятор встраивается через iframe в сайт на платформе Tilda

## 5. Система ценообразования

### 5.1. Архитектура ценообразования

Система ценообразования — самый сложный компонент приложения. Она построена на принципе **матричного поиска с экстраполяцией**.

### 5.2. Структура данных

Цены хранятся в виде вложенных словарей Python (модуль `integrated_pricing.py`):

```
Модель → Тип конструкции → Ширина → Глубина → Цена (EUR)
```

**Пример структуры:**
```python
PROJECT_LIGHT_WALL_PRICES = {
    "3.0": {   # ширина 3 метра
        "3.0": 2454.0,   # глубина 3м → 2454 EUR
        "3.5": 2577.0,   # глубина 3.5м → 2577 EUR
        "4.0": 2772.0,   # глубина 4м → 2772 EUR
        ...
    },
    "3.5": { ... },
    "4.0": { ... },
    ...
}
```

### 5.3. Модели пергол

| Модель | Описание | Макс. ширина модуля |
|--------|----------|---------------------|
| Project & Light | Базовая модель | 4.5 м |
| Basic & Pro New | Расширенная модель | 4.5 м |
| FLY | Консольная модель | Специальные правила |

### 5.4. Система модулей

Широкие перголы разбиваются на модули:
- **1 модуль:** ширина до 4.5 м
- **2 модуля:** ширина 4.5–9.0 м
- **3 модуля:** ширина 9.0–13.5 м
- **4 модуля:** ширина 13.5–16.0 м

**Логика определения количества модулей:**
```python
def determine_module_count(width, model):
    if width <= 4.5:
        return 1
    elif width <= 9.0:
        return 2
    elif width <= 13.5:
        return 3
    else:
        return 4
```

### 5.5. Экстраполяция цен

Когда точная комбинация ширина×глубина отсутствует в таблице, применяется **линейная интерполяция** между ближайшими известными значениями:

```
Цена = ЦенаНижняя + (ЦенаВерхняя - ЦенаНижняя) × (Размер - РазмерНижний) / (РазмерВерхний - РазмерНижний)
```

### 5.6. Наценки и надбавки

| Тип наценки | Значение | Условие |
|------------|----------|---------|
| Нестандартный цвет | +10% | Цвет не RAL 9016 (белый) |
| Автоматика | Фиксированная сумма | Дистанционное управление |
| Монтаж | Рассчитывается | По площади и сложности |
| Конвертация EUR→RUB | ×110 | Курс EXCHANGE_RATE_EUR_RUB |

### 5.7. Алгоритм расчета (поэтапно)

```
1. Получить входные данные (тип, модель, ширина, глубина, цвет)
      ↓
2. Определить количество модулей по ширине
      ↓
3. Найти базовую цену в матрице (или экстраполировать)
      ↓
4. Умножить на количество модулей (если > 1)
      ↓
5. Применить наценку за цвет (+10% для нестандартного)
      ↓
6. Добавить стоимость опций (автоматика, освещение)
      ↓
7. Конвертировать из EUR в RUB (×110)
      ↓
8. Добавить стоимость монтажа (в RUB)
      ↓
9. Вернуть итоговую стоимость
```

## 7. Интеграция с Tilda (iframe)

### 7.1. Принцип встраивания

Калькулятор размещается на сайте Tilda через HTML-блок с `<iframe>`:

```html
<!-- Код в блоке Tilda T123 -->
<iframe 
    id="pergola-calculator"
    src="https://your-app.replit.app/embed"
    style="width: 100%; border: none;"
    scrolling="no">
</iframe>
```

### 7.2. Система автоподстройки высоты (Resize Messenger)

Ключевая проблема iframe — фиксированная высота, не соответствующая контенту. Решение реализовано через `postMessage API`:

**Внутри iframe (калькулятор):**
```javascript
// resize_messenger.js
function sendHeight() {
    const height = document.body.scrollHeight + 30; // запас 30px
    window.parent.postMessage({
        type: 'resize',
        height: height
    }, '*');
}

// Мониторинг каждые 2 секунды
setInterval(sendHeight, 2000);

// Отслеживание изменений DOM
const observer = new MutationObserver(sendHeight);
observer.observe(document.body, { childList: true, subtree: true });
```

**На стороне Tilda (родительская страница):**
```javascript
window.addEventListener('message', function(event) {
    if (event.data.type === 'resize') {
        document.getElementById('pergola-calculator')
            .style.height = event.data.height + 'px';
    }
});
```

### 7.3. Учет состояний калькулятора

Высота автоматически пересчитывается при:
- Развертывании/свертывании дополнительных опций
- Появлении результатов расчета
- Отображении ошибок валидации
- Изменении типа конструкции (разные изображения)

### 7.4. Маршрут /embed

Специальный маршрут для iframe-версии без навигации и внешних элементов:
```python
@app.route('/embed')
def embed():
    return render_template('embed.html')
```

## 8. База данных

### 8.1. PostgreSQL

**Подключение:** через переменную окружения `DATABASE_URL`  
**ORM:** SQLAlchemy (через Flask-SQLAlchemy)

```python
app.config["SQLALCHEMY_DATABASE_URI"] = os.environ.get("DATABASE_URL")
app.config["SQLALCHEMY_ENGINE_OPTIONS"] = {
    "pool_recycle": 300,      # Переподключение каждые 5 минут
    "pool_pre_ping": True,    # Проверка соединения перед использованием
}
```

### 8.2. Модель данных

```python
class User(UserMixin, db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(64), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    password_hash = db.Column(db.String(256))
```

### 8.3. Ценовые данные

Ценовые данные хранятся **не в БД**, а в:
- Модуле Python `integrated_pricing.py` (хардкод-словари)
- JSON-файле `static/data/integrated_pricing.json`
- CSV-файлах в `attached_assets/`

Это обеспечивает мгновенный доступ без запросов к БД и возможность горячей перезагрузки через `/reload-prices`.

## 9. Архитектурные принципы

### 9.1. Монолитная архитектура

Все компоненты (маршрутизация, бизнес-логика, рендеринг, PDF) в одном приложении. Преимущества для данного проекта:
- Простота развертывания (один процесс)
- Отсутствие сетевых задержек между сервисами
- Простая отладка

### 9.2. Разделение ответственности (Separation of Concerns)

Несмотря на монолитность, код разделен на логические модули:

```
app.py                    — Маршрутизация и координация
integrated_pricing.py     — Данные о ценах
simple_pdf_generator.py   — Генерация PDF
reportlab_pdf_generator.py — Альтернативный PDF-генератор
static/js/calculator.js   — Логика формы
static/js/displayResults.js — Отображение результатов
static/js/validation.js   — Валидация данных
```

### 9.3. Data-Driven Design (управление через данные)

Цены, лимиты размеров, наименования типов конструкций — всё вынесено в данные, а не захардкожено в логику:

```python
CONSTRUCTION_TYPES = {
    'wall': 'Пристенная пергола',
    'standalone': 'Отдельностоящая пергола',
    'twin': 'Двухскатная пергола',
    'wall_fly': 'Пристенная консольная пергола FLY',
    'solo_fly': 'Отдельностоящая консольная пергола FLY',
    'twin_fly': 'Двухскатная консольная пергола FLY'
}
```

### 9.4. Отказоустойчивость (Fault Tolerance)

Многоуровневая защита от сбоев:
- **PDF генерация:** Основной генератор → запасной генератор → текстовый отчет
- **Загрузка шрифтов:** Локальные файлы → системные шрифты → стандартный Helvetica
- **Ценовые данные:** JSON → CSV → хардкод в Python-модуле
- **Все критические блоки** обернуты в `try-except` с информативным логированием

```python
try:
    pdf_buffer = reportlab_pdf_generator.generate_pdf(data)
except Exception as e:
    logging.error(f"Ошибка основного генератора: {e}")
    try:
        pdf_buffer = simple_pdf_generator.generate_pdf(data)
    except Exception as e2:
        logging.error(f"Ошибка запасного генератора: {e2}")
        return jsonify({"error": "Не удалось сгенерировать PDF"}), 500
```

### 9.5. Defensive Programming (защитное программирование)

```python
# Безопасное получение данных с значениями по умолчанию
width = safe_float(data.get('width', 3.0))
depth = safe_float(data.get('depth', 3.0))
color = data.get('color', 'RAL 9016')

# Валидация диапазонов
if width < 2.0 or width > 16.0:
    return jsonify({"error": "Ширина должна быть от 2 до 16 метров"}), 400
```

### 9.6. Кэширование

Применяется декоратор `@lru_cache` для кэширования результатов повторных вычислений:
```python
from functools import lru_cache

@lru_cache(maxsize=128)
def get_cached_price(model, construction_type, width, depth):
    return calculate_price(model, construction_type, width, depth)
```

### 9.7. Логирование

Двухуровневая система логирования (файл + консоль):
```python
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(levelname)s - %(message)s',
    handlers=[
        logging.FileHandler('app_debug.log'),
        logging.StreamHandler()
    ]
)
```

## 12. Безопасность

### 12.1. Реализованные меры

| Мера | Реализация |
|------|------------|
| Секретный ключ сессии | `app.secret_key = os.environ.get("SESSION_SECRET")` |

| CORS-политика | Flask-CORS с настроенными разрешенными источниками |
| Валидация входных данных | Серверная проверка всех параметров расчета |
| SQL Injection защита | ORM SQLAlchemy (параметризованные запросы) |
| Безопасное преобразование типов | Функция `safe_float()` для числовых данных |

### 12.2. Хранение секретов

Все чувствительные данные хранятся в переменных окружения Replit (Secrets), а не в коде.
