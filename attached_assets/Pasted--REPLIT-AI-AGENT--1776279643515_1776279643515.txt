### Логика
Счётчик — это социальное доказательство. Он должен выглядеть как живой,
регулярно обновляемый показатель, а не статичная цифра.

### Стартовое значение
- Базовое число: 10 (уже показывается на сайте)
- Базовая дата: 01.01.2026

### Формула автоинкремента
Каждую неделю прибавляем +1 к счётчику.
Формула:
  weeks_passed = (today - base_date).days // 7
  current_count = base_count + weeks_passed

Пример: если сегодня 15.04.2026, то прошло ~15 недель → счётчик = 25.

### Реализация
1. Создай helper-функцию get_pergola_count() в utils.py:

   from datetime import date
   def get_pergola_count():
       base_date = date(2026, 1, 1)
       base_count = 10
       weeks = (date.today() - base_date).days // 7
       return base_count + weeks

2. Передавай это значение в ВСЕ шаблоны где показывается счётчик:
   - index.html / calculator.html (зелёный баннер)
   - kp.html (страница КП)
   - pdf_template.html

3. В шаблоне используй: {{ pergola_count }} ПЕРГОЛ установлено в {{ current_year }} году

### НЕ НУЖНО
- Cron-задачи, celery, APScheduler — пересчёт происходит при каждом запросе
- База данных для счётчика — только вычисление на лету

Завод-производитель — Decolife (decolife.pro). Нужно один раз при запуске
(или по команде) скачать с их сайта изображения и описания для каждой
модели перголы и сохранить локально. Это даст качественный визуальный
контент для КП.

B700 (поворотно-сдвижные):
  https://decolife.pro/products/pergolas/bioclimatic-pergolas/bioclimatic-with-sliding-lamellas/

B600 (PIR сэндвич-панели):

### Что именно парсить на каждой странице
Для каждой модели сохранить в JSON-файл:
  {
    "model": "B500",
    "title": <заголовок страницы H1>,
    "description": <первый абзац описания или мета-описание>,
    "features": [<список преимуществ/характеристик — теги li из основного блока>],
    "images": [

        "url": <абсолютный URL изображения>,
        "alt": ,
        "local_path": "static/decolife/b500/img_0.jpg"
      },
      ...
    ]
  }

### Какие изображения брать
- Только изображения из основного контентного блока страницы (не шапка, не меню)
- Приоритет: img с классами типа .product-image, .gallery, .slider — смотри реальную структуру
- Максимум 6 изображений на модель
- Минимальный размер: пропускай картинки < 200×200px (иконки и декор)
- Форматы: .jpg, .jpeg, .png, .webp
- Скачивать с заголовком User-Agent: Mozilla/5.0 (чтобы не блокировали)
