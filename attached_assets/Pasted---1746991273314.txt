Архитектура и паттерны приложения калькулятора должно быть  построено по монолитной архитектуре с использованием Flask на бэкенде и ванильного JavaScript на фронтенде. Вот подробное описание архитектуры и используемых паттернов:
Архитектура приложения
1. Серверная часть (Backend):
    * Flask - легковесный веб-фреймворк на Python
    * PostgreSQL - реляционная база данных для хранения расчетов и кэширования
    * Gunicorn - WSGI-сервер для запуска Flask-приложения в production-окружении
2. Клиентская часть (Frontend):
    * Vanilla JavaScript - чистый JavaScript без фреймворков
    * Bootstrap - CSS-фреймворк для стилизации интерфейса
    * HTML Templates (Jinja2) - шаблонизатор для рендеринга страниц
3. PDF-генерация:
    * ReportLab - библиотека для создания PDF-документов
    * PyPDF2 - для работы с существующими PDF-файлами
Применяемые паттерны разработки
1. MVC (Model-View-Controller):
    * Model: Логика обработки данных и расчетов цен перголы
    * View: HTML-шаблоны и JavaScript для рендеринга пользовательского интерфейса
    * Controller: Маршруты Flask, обрабатывающие запросы
2. REST API:
    * Эндпоинты для сохранения и получения расчетов
    * JSON-формат для обмена данными между клиентом и сервером
3. Кэширование:
    * In-memory кэширование результатов расчета для повышения производительности
    * Хранение популярных конфигураций в кэше
4. Модули и разделение ответственности:
    * PDF-генерация выделена в отдельные модули (simplified_pdf_generator.py, simple_pdf_generator.py)
    * Клиентский код разделен на модули по функциональности (UI, валидация, PDF-экспорт)
5. Фабричный метод:
    * Для создания различных конфигураций пергол в зависимости от типа и параметров
6. Стратегия:
    * Для разных алгоритмов расчета цен в зависимости от типа перголы и модели
Ключевые технические особенности
1. Интернационализация:
    * Поддержка кириллицы в PDF-документах
    * Корректная обработка Unicode при экспорте файлов
2. Адаптивный дизайн:
    * Поддержка различных устройств (PC, tablet, mobile)
    * Оптимизация интерфейса для Tilda CMS через iframe-интеграцию
3. Оптимизация производительности:
    * Асинхронная загрузка данных и изображений
    * Предзагрузка изображений типов пергол
4. Обработка ошибок:
    * Подробное логирование на сервере
    * Интуитивно понятные сообщения для пользователя
5. Поддержка сложных вычислений:
    * Экстраполяция цен для больших размеров
    * Оптимизация количества конструкций для минимизации стоимости

Эволюция архитектуры
Приложение прошло несколько этапов развития:
1. v1.x: Базовый калькулятор с минимальным функционалом
2. v2.x: Добавление различных моделей пергол и опций
3. v3.x: Расширенные функции PDF-экспорта, улучшенный интерфейс, оптимизация для мобильных устройств
4. v3.11 (текущая): Улучшенное именование файлов PDF, исправления в текстах и параметрах
Такая архитектура обеспечивает хорошую поддерживаемость, легкое масштабирование функционала и надежную работу приложения при сохранении высокой производительности.
