## Установка и настройка

### Требования к окружению
- Python 3.9 или выше
- Пакеты из файла `pyproject.toml`

### Шаги установки

1. Клонируйте репозиторий:
   ```
   git clone [URL_репозитория]
   cd [название_директории]
   ```

2. Установите зависимости с помощью Poetry:
   ```
   poetry install
   ```
   
   Или с помощью pip:
   ```
   pip install -r requirements.txt
   ```

3. Запустите приложение:
   ```
   streamlit run app.py --server.port 5000
   ```

## Публикация и встраивание

### Публикация на Replit

1. Убедитесь, что приложение корректно работает локально
2. Настройте `.streamlit/config.toml` для публикации:
   ```
   [server]
   headless = true
   enableCORS = true
   enableXsrfProtection = true
   address = "0.0.0.0"
   port = 5000
   ```
3. Настройте `workflows_config.toml`:
   ```
   [[run]]
   name = "MainServer"
   command = "streamlit run app.py --server.port 5000 --server.address 0.0.0.0"
   wait_for_port = 5000
   ```
4. Используйте кнопку "Deploy" в интерфейсе Replit

### Встраивание на веб-сайт

1. Используйте код из файла `embed_code_for_tilda.html` для встраивания на сайт Tilda
2. Для других CMS адаптируйте код, сохраняя основную структуру:
   ```html
   <iframe 
       src="https://[URL_вашего_приложения]" 
       width="100%" 
       height="650px" 
       frameborder="0"
       scrolling="no"
       onload="adjustIframeHeight(this)"
   ></iframe>
   <script>
   function adjustIframeHeight(iframe) {
     // Эта функция вызывается после загрузки iframe и позволяет
     // адаптировать высоту при изменении содержимого
     window.addEventListener('message', function(e) {
       if (e.data && e.data.type === 'iframe-height') {
         iframe.style.height = e.data.height + 'px';
       }
     });
   }
   </script>
   ```

## Работа с изображениями и галереей

### Обработка изображений

1. Приложение поддерживает различные форматы изображений, включая:
   - JPEG/JPG
   - PNG
   - HEIC (автоматическая конвертация в JPEG)

2. Файлы в формате HEIC автоматически конвертируются в JPEG при загрузке:
   ```python
   # Пример использования конвертера HEIC
   from heic_converter import convert_heic_to_jpeg
   
   jpeg_path = convert_heic_to_jpeg('path/to/image.heic')
   ```

3. Система управления изображениями поддерживает:
   - Загрузку новых изображений через интерфейс администрирования
   - Активацию/деактивацию изображений для галереи
   - Удаление изображений с сохранением в архиве
   - Постоянное удаление изображений из системы

### Обнаружение и обработка дубликатов

Система реализует многоуровневую проверку дубликатов:

1. **Проверка по имени файла**: Определяет явные дубликаты
2. **Проверка по MD5-хешу**: Выявляет идентичные файлы даже с разными именами
3. **Проверка по содержимому**: Анализирует содержимое изображений для обнаружения семантических дубликатов

```python
# Пример использования модуля обнаружения дубликатов
from components.gallery_admin import detect_duplicate_images

duplicates = detect_duplicate_images('path/to/image.jpg')
if duplicates:
    print(f"Обнаружены дубликаты: {duplicates}")
```

### Кэширование хешей изображений

Для повышения производительности система использует кэширование хешей:

1. При первой загрузке изображения вычисляется его MD5-хеш
2. Хеш сохраняется в файле кэша для быстрого доступа
3. При последующих проверках используется кэшированное значение

```python
# Пример работы с кэшем хешей
from cache_manager import ImageHashCache

cache = ImageHashCache()
image_hash = cache.get_or_compute_hash('path/to/image.jpg')
```

### Групповые операции с изображениями

Администраторы могут выполнять групповые операции:

1. Массовое исключение дубликатов
2. Активация или деактивация группы изображений
3. Сохранение только одной копии из группы дубликатов

## Часто задаваемые вопросы

### Как оптимизировать скорость работы и время отклика приложения?

Для повышения скорости работы и отзывчивости приложения применяются следующие подходы:

1. Используйте кэширование вычислений с помощью декоратора `@st.cache_data`:
   ```python
   @st.cache_data
   def load_price_data(pergola_type, lamella_size):
       # Тяжелые вычисления или загрузка данных
       return data
   ```

2. Оптимизируйте скорость скроллинга к результатам с помощью трёх параллельных методов:
   ```javascript
   // Функция быстрой прокрутки с использованием мультиметодного подхода
   function scrollToResults() {
       // МЕТОД 1: Прямой скролл без анимации (самый быстрый)
       var resultsElement = document.getElementById('results');
       if (resultsElement) {
           // Используем 'auto' вместо 'smooth' для максимальной скорости
           resultsElement.scrollIntoView({behavior: 'auto', block: 'start'});
       }
       
       // МЕТОД 2: Прямая установка scrollTop с точным позиционированием
       var rect = resultsElement.getBoundingClientRect();
       var scrollTop = window.pageYOffset || document.documentElement.scrollTop;
       var targetY = rect.top + scrollTop - 60;
       window.scrollTo(0, targetY);
       
       // МЕТОД 3: Использование хэш-навигации (быстрый метод в большинстве браузеров)
       window.location.hash = 'results';
   }
   ```

3. Используйте CSS свойства для улучшения позиционирования элементов при скролле:
   ```css
   #results {
       scroll-margin-top: 60px !important;
       scroll-snap-align: start !important;
       visibility: visible !important;
   }
   ```

4. Для мгновенной визуальной обратной связи используйте быстрые анимации:
   ```javascript
   function blinkResultsHeader() {
       var header = document.getElementById('results-header');
       if (header) {
           // Используем короткие интервалы (200мс) для быстрой анимации
           header.style.transition = 'background-color 0.15s';
           header.style.backgroundColor = '#FFEB3B'; // Жёлтый для привлечения внимания
           
           setTimeout(function() {
               header.style.backgroundColor = '#e0f0ff'; // Голубой
               setTimeout(function() {
                   header.style.backgroundColor = 'white'; // Возврат к исходному
               }, 200);
           }, 200);
       }
   }
   ```

2. Добавлены специальные CSS классы для различных устройств (device-smartphone, device-tablet, device-laptop, device-desktop)
3. Созданы адаптивные стили для секций "Размеры" и "Освещение" с ультра-компактным отображением на смартфонах
4. Добавлено определение ориентации экрана для оптимизации в режимах portrait/landscape
5. Улучшена видимость кнопки "Рассчитать стоимость" на первом экране мобильных устройств
6. Все интерактивные элементы формы доступны без прокрутки на мобильных устройствах
7. Используются CSS-переменные для настройки размеров шрифтов и отступов в зависимости от устройства
8. Уменьшены размеры полей ввода на мобильных устройствах для большей компактности (padding: 0.2rem, height: 1.8rem)
9. Оптимизировано расположение опций светодиодного освещения, заменён горизонтальный выбор на вертикальный

   ```javascript
   function detectDeviceAndAddClass() {
       const body = document.querySelector('body');
       const width = window.innerWidth;
       
       // Очищаем предыдущие классы
       body.classList.remove('device-smartphone', 'device-tablet', 'device-laptop', 'device-desktop');
       
       // Определяем тип устройства
       if (width < 576) {

       } else if (width < 992) {

       } else if (width < 1400) {
           body.classList.add('device-laptop');
       } else {
           body.classList.add('device-desktop');
       }
   }

   // Запускаем при загрузке и изменении размера
   detectDeviceAndAddClass();
   window.addEventListener('resize', detectDeviceAndAddClass);
   ```

### Как изменить фирменные цвета приложения?

Отредактируйте секцию `[theme]` в файле `.streamlit/config.toml`:

```toml
[theme]
primaryColor = "#004B9A"  # Основной цвет
backgroundColor = "#FFFFFF"  # Фон
secondaryBackgroundColor = "#F0F2F6"  # Вторичный фон
textColor = "#262730"  # Цвет текста
```

Логика формирования PDF-документов находится в файле `pdf_generator_fpdf.py`. Для изменения шаблона редактируйте функцию `generate_commercial_offer()`.
